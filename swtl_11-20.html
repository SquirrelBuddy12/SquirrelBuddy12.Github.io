Smarter Way to Learn 11-20

SWTL 11

Comparison Operators

The comparison operator === is a type of equality operator. As you learned in the last chapter, you use it to compare two things to see if they're equal. You can use the it to compare a variable with a string, a variable with a number, a variable with a math expression, or a variable with a variable. And you can use it to compare various combinations.

SWTL 12

If else and else if

The if statements you've coded so far have been all-or-nothing. If the condition tested true, something happened. If the condition tested false, nothing happened. If we have several statements all bases are covered, depending on what the user enters.

SWTL 13

Testing Sets of Conditions

Using the if statement, you've learned to test for a condition. If the condition is met, one or more statements execute. However, you may have several conditions that have to be met in order for a test to succeed.

SWTL 14

If Statements Nested

Nest levels are communicated to JavaScript by the positions of the curly brackets. There are three blocks nested inside the top-level if. If the condition tested by the top-level if—that c has the same value as d—is false, none of the blocks nested inside executes. The opening curly bracket and the closing curly bracket enclose all the nested code, telling JavaScript that everything inside is second-level.

SWTL 15

Arrays

An array is a variable that can have multiple values assigned to it.

SWTL 16

Arrays with Adding and Removing Elements

You can declare an empty variable, one that doesn't have a value. Then you can assign it a value of whatever you want. And you can change its value at will. You can do all these things with an array, as well.

SWTL 17

More Arrays With Shift and Unshift

Use the shift method to remove an element from the beginning of an array. To add one or more elements to the beginning of an array, use the unshift method.

SWTL 18

For Loops

The for loop - loops through a block of code a number of times: The for loop has the following syntax: Statement 1 is executed (one time) before the execution of the code block. Statement 2 defines the condition for executing the code block. Statement 3 is executed (every time) after the code block has been executed.

SWTL 19

For Loops With Flags

There are several problems with the for loop example I gave you in the last chapter. The first problem is a potential communication problem. If a match between the user's city for example and the a list of cleanest cities is found, a confirming alert displays. But if there is no match, nothing happens. The user is left in the dark. If no match is found, we need to display an alert saying so. We do it with a flag. A flag is just a variable that starts out with a default value that you give it, and then is switched to a different value under certain conditions.

SWTL 20

For Loops Nested

This is an array of arrays. When you have an array of 3 arrays. The outer 'for' will loop through each of 1st level arrays. So the very first outer for loop you are going to grab the first inner array. In the inner loop you are going to loop through each of the 3 inner arrays one at a time. Such as 1 and 2 and then 3 and 4.

SWTL 81

Browser Control: Testing for Popup Blockers

Popup blockers are a standard feature of browsers, with some level of popup blocking usually built in as a default. JavaScript popups, and especially those that open a new page within the same website, are often tolerated by default, but you can never be sure. If popups are an essential feature of your site, you need to test whether your popups are going to be blocked. If they are, you can ask the user to disable the popup blocker for your site. To test it, you open a small test popup, and then quickly close it. This will probably happen so fast that the user may won't notice. If the attempt succeeds, the window handle will have a value. If the popup is blocked,
the handle will be null, your message will be delivered.

Exceptions: Try and Catch

The keyword alert, mispelled, breaks the code. But JavaScript doesn't tell you what's wrong. Of course, in the small piece of code above, it's easy to spot the problem. But what if the function runs 40 lines and calls three other functions? It would be nice if JavaScript reported the specific problem to you instead of just breaking. That's what try and catch is for.
